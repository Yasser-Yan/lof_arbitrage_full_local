# LOF套利监控用户使用说明

## 1. 软件用途

LOF套利监控是一个本地运行的小工具，用来集中查看 LOF 基金的场内行情、基金净值和申购赎回状态，辅助观察场内价格相对基金净值的溢价或折价。

软件主要显示这些信息：

- 基金代码、名称和分类
- 现价日期、场内现价、涨跌幅、成交额和成交量
- 基金净值、净值日期和溢价率
- 申购状态、赎回状态和部分限额信息
- 数据来源和刷新状态

溢价率公式：

```text
(场内现价 - 基金净值) / 基金净值 * 100%
```

溢价率为正，表示场内现价高于基金净值；溢价率为负，表示场内现价低于基金净值。

## 2. 发布文件说明

最新版本在 `发布` 文件夹中：

```text
发布\LOF套利监控安装包.exe
发布\LOF套利监控.exe
发布\funds_config.json
发布\配置文件说明.txt
```

各文件用途：

| 文件 | 用途 |
| --- | --- |
| `LOF套利监控安装包.exe` | 推荐给普通用户使用。会安装程序、配置文件，并创建桌面和开始菜单快捷方式。 |
| `LOF套利监控.exe` | 绿色单文件版本。适合临时运行或放在 U 盘/指定目录运行。 |
| `funds_config.json` | 基金配置和最近一次抓取的数据缓存，包括基金池、价格、净值、申购状态等。 |
| `配置文件说明.txt` | 简要说明配置文件放在哪里、如何迁移。 |

## 3. 安装方式

### 方式一：安装包安装，推荐

1. 打开 `发布` 文件夹。
2. 双击 `LOF套利监控安装包.exe`。
3. 安装完成后，软件会写入到：

```text
%LOCALAPPDATA%\Programs\LOF套利监控
```

4. 安装程序会创建桌面快捷方式和开始菜单快捷方式。
5. 安装完成后一般会自动启动软件。

如果安装目录里已经存在 `funds_config.json`，安装包不会覆盖旧配置，而是把新默认配置保存为：

```text
funds_config_default.json
```

这样可以保护你已有的自定义基金池和最近数据。

### 方式二：绿色版直接运行

1. 打开 `发布` 文件夹。
2. 确保 `LOF套利监控.exe` 和 `funds_config.json` 在同一个目录。
3. 双击 `LOF套利监控.exe`。
4. 程序会自动打开浏览器页面。

如果浏览器没有自动打开，可以手动访问：

```text
http://127.0.0.1:8787
```

## 4. 启动和退出

### 启动

双击桌面快捷方式、开始菜单快捷方式，或直接双击 `LOF套利监控.exe`。

如果软件已经在后台运行，再次双击程序会打开已有服务的网页，不会重复启动多个服务。

### 退出

网页右上角有 `退出程序` 按钮。点击后，后台本地服务会退出。

只关闭浏览器页面不会结束后台服务。如果要彻底关闭，请使用页面里的 `退出程序`。

## 5. 页面功能说明

### 顶部按钮

| 按钮 | 作用 |
| --- | --- |
| `快速刷新` | 只刷新场内行情，速度较快。会更新现价、现价日期、涨跌幅、成交额、成交量，并重新计算溢价率。 |
| `全量刷新` | 重新抓取基金池、净值、申购赎回状态和场内行情。速度较慢，但信息最完整。 |
| `导出CSV` | 导出当前后端保存的全部基金数据。 |
| `退出程序` | 关闭本地服务。 |
| `自动刷新` | 开启后每 30 秒快速刷新一次行情。 |

### 分类标签

页面分为四类：

- 股票LOF
- 指数LOF
- 欧美市场
- 亚洲市场

QDII 基金会按名称特征分到欧美市场或亚洲市场。分类只用于观察和筛选，不代表投资属性的完整判断。

### 搜索/添加

搜索框支持输入基金代码或基金名称。

如果基金已经在列表中，会自动定位到对应分类。

如果基金不在当前列表中，点击 `搜索/添加` 会查询基金代码库，判断是否符合场内 LOF 条件；符合条件时会加入列表，并尝试补齐净值、行情和交易状态。

## 6. 表格字段说明

| 字段 | 说明 |
| --- | --- |
| 代码 | 基金代码。 |
| 名称 | 基金名称。 |
| 现价日期 | 场内行情对应的交易日期。优先使用东方财富行情接口返回的交易日期。 |
| 现价 | 场内最新价格。 |
| 涨跌幅 | 场内价格相对上一交易日收盘价的涨跌幅。 |
| 净值 | 基金单位净值。 |
| 净值日期 | 基金净值对应日期。QDII 净值通常会滞后。 |
| 溢价率 | 根据场内现价和基金净值计算。 |
| 成交额(万) | 场内成交额，单位为万元。 |
| 成交量 | 场内成交量。 |
| 申购 | 当前申购状态，例如开放申购、暂停申购、限大额。 |
| 赎回 | 当前赎回状态，例如开放赎回、暂停赎回。 |
| 来源 | 该基金记录的主要来源。 |

提示：

- `现价日期` 为空，通常表示该份额没有拿到实时场内行情，例如部分后端份额或无行情返回的份额。
- `净值日期` 和 `现价日期` 不一定相同，QDII 基金尤其常见。
- `申购` 显示为限大额时，页面会尽量显示限额，例如 `100.00元` 或 `100.00万元`。

## 7. 排序和查看

点击任意表头可以排序。

常用排序方式：

- 点击 `溢价率`：查看溢价或折价较大的基金。
- 点击 `现价日期`：查看行情日期是否最新。
- 点击 `申购`：集中查看暂停申购、限大额或开放申购的基金。
- 点击 `成交额(万)`：查看成交活跃度。

## 8. 数据刷新建议

推荐使用方式：

1. 每天首次打开后，先点一次 `全量刷新`。
2. 交易时间内观察价格变化时，使用 `快速刷新` 或打开 `自动刷新`。
3. 如果关注申购限制变化，使用 `全量刷新`，因为申购赎回状态不会在快速刷新中逐只重抓。

全量刷新会访问多个公开数据页面，速度可能受网络和数据源限制影响。如果页面提示仍在刷新，可以稍等后再看。

## 9. 配置文件和数据迁移

配置文件名：

```text
funds_config.json
```

安装版默认位置：

```text
%LOCALAPPDATA%\Programs\LOF套利监控\funds_config.json
```

绿色版位置：

```text
LOF套利监控.exe 同目录\funds_config.json
```

换电脑时，把旧电脑的 `funds_config.json` 复制到新电脑同一安装目录，覆盖同名文件即可。

如果配置文件损坏或想恢复默认基金池，可以：

1. 退出程序。
2. 备份当前 `funds_config.json`。
3. 用发布目录里的 `funds_config.json` 或安装目录里的 `funds_config_default.json` 替换当前配置。
4. 重新启动程序。

## 10. 常见问题

### 双击后浏览器没打开

手动访问：

```text
http://127.0.0.1:8787
```

如果仍打不开，可能是程序没有启动成功，或 8787 端口被其他程序占用。

### 关闭网页后程序还在运行

这是正常现象。网页只是显示界面，后台本地服务仍在运行。

要彻底退出，请点击页面右上角的 `退出程序`。

### 现价日期为空

一般表示没有拿到该基金份额的实时场内行情。常见于后端收费份额、部分 C 类份额或行情源不返回的代码。

软件不会用当前日期硬填，避免造成误判。

### 申购状态和实际交易平台不一致

申购赎回状态来自公开页面抓取，可能有延迟，也可能与不同销售渠道显示不完全一致。实际交易前请以基金公司、销售平台和交易所信息为准。

### 净值日期比现价日期旧

这是正常情况。基金净值通常在交易日后披露，QDII 基金还可能因为海外市场和节假日产生更长滞后。

### 全量刷新比较慢

全量刷新需要抓基金池、净值、交易状态和行情。申购赎回状态是逐只基金访问档案页补齐的，所以会比快速刷新慢。

## 11. 风险提示

本工具只做数据整理和套利观察，不构成投资建议。LOF 套利涉及场内成交、申购赎回限制、净值滞后、交易费用、到账周期、限购、停牌和市场波动等风险。实际操作前请自行核验数据并评估风险。
